In the last session we looked at Primitive Data Types such as Integer, String, Boolean and Decimal. We also learned about the difference between statements and expressions. Today we are going to focus our attention on a special component of statements called variables.

Depending on the programming language a variable consists of at least two distinct parts: a name and a value. However, in languages like Apex, Java or C, variables consist of three distinct parts: a name, a value and a type.

Let's take a look at what variable assignment statements look like in Apex:

{% highlight ruby linenos %} /* Integer variables */ Integer number = 1; Integer age = 157; /* String variables */ String favoriteBand = 'Jackson 5'; String daBoss = 'Adam Robinson'; /* Decimal variables */ Decimal quarterlyRevenue = 999999.99; Decimal averageGPA = 3.5; /* Boolean variables */ Boolean truthy = true; Boolean falsey = false; /* Date variables are special: */ Date mahBday = Date.newInstance(1982, 04, 03); Date nextNewYear = Date.newIstance(2016, 01, 01); /* Time variables are also special: */ Time lunchTime = Time.newInstance(12, 0, 0, 0); Time dinnerTime = Time.newInstance(19, 0, 0, 0); /* If Date variables are special, and Time variables are special, Then DateTime variables must be special, too. */ DateTime lastLogin = DateTime.newInstance(2015, 06, 24, 10, 0, 0); {% endhighlight %}

Breaking It Down

Let's break down a single variable assignment statement to understand the components. Given the following statement:

{% highlight ruby linenos %} Integer number = 1; {% endhighlight %}

The first component of the statement is the type. In this example, we are declaring an Integer variable:

{% highlight ruby linenos %} /* The first component of a variable assignment statement is the type (Integer) */ Integer number = 1; {% endhighlight %}

The second component of this statement is the name of the variable, number. One of the most difficult aspects of programming is naming variables well, something we will talk more about later. The name is really not important for the computer and is used to help humans understand what information variables are holding. In this instance we are creating a variable to hold an Integer, so we use the generic name: number:

{% highlight ruby linenos %} /* The second component of a variable assignment statement is the name (number) */ Integer number = 1; {% endhighlight %}

The final component is the value we want to assign the Integer variable number with, which in this case is 1. As the term variable implies, the value of a variable can change while a program is running. This is mainly why variables are called variables - because their values can vary over time. We are not changing the value of number just yet. For now, we are simply assigning its value:

{% highlight ruby linenos %} /* The final component of a variable assignment statement is the value (1) */ Integer number = 1; {% endhighlight %}

Lastly, as part of the syntax requirements of Apex, we conclude this variable assignment statement with a ; (semicolon) to indicate the end of the statement:

{% highlight ruby linenos %} /* Don't forget your ; character at the end of each statement. Otherwise it is a syntax error. */ Integer number = 1; {% endhighlight %}

For the remainder of the session we are going to spend time in the Execute Anonymous tab of the Eclipse IDE. This will allow us to execute Apex code without any complicated setup, nor needing to worry about project structure for now. We are now going to practice writing variable assignment statements and executing them to ensure our syntax is correct. We will practice together and will cover the following:

  1. Create an Integer variable named oneHundred whose value is 100.
  2. Create an Integer variable named twoHundred whose value is 200.
  3. Create a String variable named firstName whose value is your first name.
  4. Create a String variable named lastName whose value is your last name.

The final result should look like this:

{% highlight ruby linenos %} Integer oneHundred = 100; Integer twoHundred = 200; String firstName = 'Rick'; String lastName = 'Winfrey'; {% endhighlight %}

Variables From Other Variables

So far, we have declared variables using simple values. Declaring variables in the real world however is often more interesting, especially when we declare a new variable whose value is a calculation, operation or expression of existing variables and their values. To illustrate this, let's practice declaring variables whose values are the expression of other variables:

  1. Create an Integer variable named threeHundred whose value is the sum of oneHundred and twoHundred.
  2. Create a String variable named fullName whose value is the concatenation of firstName and lastName.

The final result should look like this:

{% highlight ruby linenos %} Integer oneHundred = 100; Integer twoHundred = 200; String firstName = 'Rick'; String lastName = 'Winfrey'; Integer threeHundred = oneHundred + twoHundred; String fullName = firstName + lastName; {% endhighlight %}

Debugging

At this point, you might be wondering how it is possible to verify that threeHundred and fullName have the correct values? Fortunately, Apex gives us the facility to debug our variables' values by using a special method:

{% highlight ruby linenos %} Integer sum = 10 + 100; System.debug(sum); {% endhighlight %}

We won't worry about what System means just yet, but when we use the debug method, it prints the value of whatever variable we pass to it in the User Debug Logs:

Returning back to our variables threeHundred and fullName, let's see how we can apply the debug method to those variables and verify their values are correct:

{% highlight ruby linenos %} Integer oneHundred = 100; Integer twoHundred = 200; String firstName = 'Rick'; String lastName = 'Winfrey'; Integer threeHundred = oneHundred + twoHundred; String fullName = firstName + lastName; System.debug(threeHundred); System.debug(fullName); {% endhighlight %}

The Only Constant is Change

Previously, we stated that variables' values can change as a program is running. To help illustrate how this works, let's walk through the following steps:

  1. Create an Integer variable named yolo whose value is 1.
  2. Write a debug statement to output the value of yolo and verify that the value is 1.
  3. Reassign the value of yolo with the number 2.
  4. Write another debug statement to output the value of yolo and verify that the value is now 2.
  5. Finally, Reassign the value of yolo back to its original value of 1.
  6. Write one last debug statement to verify the value of yolo is its original value.

The final result should look like this:

{% highlight ruby linenos %} Integer yolo = 1; System.debug(yolo); yolo = 2; System.debug(yolo); yolo = 1; System.debug(yolo); {% endhighlight %}

You might be wondering why we do not require the type information on lines 5 or 9. The answer is because we declared the variable on line 1, and after that, we are free to change its value with a more simple assignment statement like those on line 5 and 9. The takeaway is that the type information is only required when we first declare a variable, but assignment statements using a previously declared variable do not need to specify the type, as it is already known.
